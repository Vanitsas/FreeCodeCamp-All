# Debug Type Coercion Errors in a Buggy App

A beginner JavaScript lab completed as part of the [freeCodeCamp](https://www.freecodecamp.org/) curriculum.

## 📌 About

In this lab, type coercion bugs in a broken app are identified and fixed. The original code contained string values where numbers were expected, causing incorrect results due to JavaScript's automatic type coercion. All variables were corrected to use proper number types.

## 🧠 Concepts Practiced

- Understanding JavaScript type coercion
- Difference between number and string values in arithmetic operations
- Debugging and fixing type-related bugs
- Using `const` for variable declarations
- Template literals with `${}`

## 💻 How to Run

1. Copy the code into your browser's developer console, or
2. Run it with Node.js:

```bash
node index.js
```

## 📄 Expected Output

```
5 + 10 = 15
8 - 5 = 3
1 + 5 = 6
0 + 8 = 8
10 * 2 = 20
0 + 22 = 22
```
